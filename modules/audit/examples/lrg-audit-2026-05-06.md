# LRG Audit — 2026-05-06 (Canonical Example)

First production use of the audit module on Levi Rodgers Real Estate Group.

## Input

- **Site:** lrgrealty.com (WPE install: lrgrealtyblog)
- **Published posts:** 1,096
- **GSC export:** 90-day window (2026-02-06 to 2026-05-06)
- **GSC tracked URLs:** 1,704

## Results

| Category | Count |
|----------|-------|
| Delete candidates (zero traffic, EN only) | 224 |
| Meta refresh candidates (impressions >= 200, CTR < 2.5%) | 381 |
| Priority rewrites (page 2-3, impressions >= 500) | 67 |
| Slug issues (date-prefix + hash suffix) | 439 in WP, 42 in GSC |
| Near-duplicate pairs | 186 (English only) |
| Exact title duplicates | 5 groups |

### Triage Classification

| Tier | Count | Description |
|------|-------|-------------|
| A | 83 | Already in lrgArticleKit format — CSS migration only |
| B | 1,013 | Legacy format — full content rewrite needed |

### Language Breakdown

- English posts (in scope): ~904
- Spanish posts (excluded): 192

## Key Findings

1. **Squarespace migration artifacts dominate:** 449 posts have date-prefixed
   slugs from the Squarespace import. However, only 42 of those appear in
   GSC data — the rest were never indexed.

2. **Slug detection vs GSC ground truth:** The WP database showed 449 slug
   issues, but GSC cross-reference revealed only 42 actually serve traffic.
   Lesson: always validate WP-side findings against GSC ground truth.

3. **Top meta refresh target:** `/lrg-blog/top-5-neighborhoods-to-live-in-austin-tx`
   had 493K impressions at 0.09% CTR — massive opportunity for title/meta
   optimization alone.

4. **Page 2-3 opportunities:** 67 pages ranking positions 11-30 with 500+
   impressions. Moving these to page 1 could capture significant traffic.

## Commands Used

```bash
# Full audit:
bash modules/audit/tools/run-audit.sh \
    --site lrg \
    --skip-gsc-pull \
    --gsc-export ~/Downloads/lrgrealty.com-Performance-on-Search-2026-05-06.zip \
    --output-dir ~/lrg-rewrite/audits/

# Individual tools:
python3 modules/audit/tools/identify-deletes.py \
    --inventory-csv audits/all-posts.csv \
    --gsc-pages-csv audits/gsc-pages-2026-05-06.csv \
    --output-csv audits/delete-candidates.csv

python3 modules/audit/tools/identify-meta-candidates.py \
    --gsc-pages-csv audits/gsc-pages-2026-05-06.csv \
    --min-impressions 200 --max-ctr 0.025 \
    --output-csv audits/meta-candidates.csv
```

## Lessons Learned

1. Always cross-reference WP database against GSC ground truth. The database
   alone over-counts issues (449 slug issues detected, only 42 real).

2. Spanish content should be a separate audit pass — mixing languages in
   the same run creates noise in duplicate detection.

3. The `--skip-inventory` flag is valuable for re-running analysis without
   re-pulling from the server.
