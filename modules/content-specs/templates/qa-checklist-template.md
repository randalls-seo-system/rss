# {{CLIENT_NAME}} — Content QA Checklist

## 0. Page Structure (MANDATORY — fail if missing)

- [ ] Page has ATF hero section
- [ ] ATF has exactly 1 H1 — no H1 in body content
- [ ] ATF has breadcrumb navigation
- [ ] ATF has topic label/eyebrow
- [ ] ATF has jump-link pills to main content sections
- [ ] ATF has hero lead paragraph (topic-specific, not generic)
- [ ] ATF has primary CTA
- [ ] ATF has 3-5 quick-reference cards with topic-specific content
- [ ] ATF has 3-5 FAQ items
- [ ] Main content is wrapped in proper container
- [ ] Main content body is 1,600+ words

## 1. Output Format

- [ ] Output is clean HTML
- [ ] No commentary outside the HTML
- [ ] No H1 in body
- [ ] H2s are Title Case
- [ ] Every table wrapped in scroll container

## 2. Opening Structure

- [ ] First H2 is "The Bottom Line Up Front"
- [ ] Bold opening verdict block exists
- [ ] Opening reads like an expert giving the verdict, not a blog intro

## 3. Article Content

- [ ] Every H2 begins with a direct 1-2 sentence answer (30 words or less)
- [ ] H2 answer followed by supporting text to fill 50-70 word total intro
- [ ] Each H2 section contains 3-4 substantive items (18+ words per item)
- [ ] Bullets for features/facts, numbered lists for processes/steps
- [ ] At least one comparison table (when applicable)
- [ ] Minimum 3 callouts using approved titles
- [ ] Minimum 2 CTA pills distributed through body
- [ ] External links use descriptive source text

## 4. Voice

- [ ] Sounds like an experienced professional
- [ ] Direct, practical, specific
- [ ] Confident but not theatrical
- [ ] No generic blog tone
- [ ] No content-marketer phrasing

## 5. Forbidden Language

- [ ] Does not mention generic content or other sites
- [ ] Does not use banned phrases from voice guide
- [ ] Does not sound like it's trying to sound tough
- [ ] Does not overuse qualifiers

## 6. Data-First Rule

- [ ] Each section includes at least one useful number, threshold, or data point
- [ ] No random stuffed numbers
- [ ] Key thresholds surfaced where relevant

## 7. Internal Linking

- [ ] Parent hub linked within first 25% of content
- [ ] Relevant tool pages linked at first contextual mention
- [ ] No self-links
- [ ] No duplicate destination URLs
- [ ] No links in restricted zones (callouts, tables, FAQ answers)
- [ ] Anchor text is 3-12 words, descriptive
- [ ] No bare acronyms as anchor text
- [ ] Internal links >= external links in count

## 8. Closing Sections

- [ ] "The Bottom Line" section with bold verdict + context
- [ ] FAQ section with direct, borrower-facing answers
- [ ] Resources Used section with clean source citations
- [ ] Resources Used is last section

## 9. Capitalization

- [ ] {{PROPER_NOUN_1}} always capitalized in prose
- [ ] {{PROPER_NOUN_2}} always capitalized in prose
- [ ] URLs, attributes, code blocks NOT modified
